# Scenario Template

Every evaluation scenario in this framework follows the same structure to ensure consistency, repeatability, and comparability across different AI agents and evaluation runs.

## 1. Description

A brief summary of what this scenario is designed to evaluate and why it exists.

## 2. User Prompt

The exact prompt or request provided to the AI agent.

## 3. User Constraints

All explicit requirements, preferences, and limitations that the agent must satisfy.

## 4. Expected Behaviour

A high-level description of what a successful response should accomplish. This defines the expected outcome without prescribing a specific implementation.

## 5. Evaluation Criteria

The evaluation dimensions that apply to this scenario, along with the success criteria for each dimension.

## 6. Pass Criteria

The minimum conditions required for the scenario to be considered successful.

## 7. Failure Conditions

Specific behaviors, omissions, or violations that should cause the evaluation to fail or receive a significant penalty.

## 8. Notes

Additional context, assumptions, edge cases, or future improvements related to the scenario.
